# Hash Tables

## What You Will Learn

- How hash tables work (hash function, buckets, collision handling)
- O(1) average-case lookup, insertion, deletion
- When to use hash maps vs hash sets
- Complement search pattern (Two Sum foundation)
- Frequency counting and duplicate detection

## Why This Matters

Hash tables are the most important data structure for algorithm problems. They transform O(n) lookups into O(1), enabling solutions that would otherwise require nested loops. The complement search pattern — "for each element, does its complement exist?" — is the key to dozens of problems.
