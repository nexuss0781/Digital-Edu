Content Types

Seven learning formats, one consistent system

Each content type is authored as a Markdown file with YAML front-matter. Add a file to the curriculum directory, capture the structure, and it becomes available immediately — no database migrations, no configuration changes.

Note

Rich Markdown articles with optional embedded inline quizzes. Read, comprehend, and validate understanding within a single document.

Quiz

One question per page with immediate correct or incorrect feedback. Designed for quick understanding checks between topics.

Test

Five questions per page with aggregated results and configurable error-budget pass logic. Standardised knowledge assessment.

Exam

Ten questions per page for high-stakes qualification. Built on the same assessment engine as quizzes and tests, with configurable pass thresholds.

Workshop

Step-by-step guided exercises with exact-match command validation. Progress persists across sessions, enabling self-paced practice.

Practical

Full integrated development environment with requirements checklist, Monaco code editor, and live iframe preview. Write and test code in the browser.

Project

IDE-based project submission with instructor-reviewed verdict workflow. Submit code, receive Passed or Retry feedback, and iterate until mastery.

Assessment Engine

Three modes, one unified engine

The same battle-tested question format scales across every assessment mode. Pass logic uses a configurable error budget or percentage threshold with sensible defaults. Type normalisation handles case, plurals, and common typos automatically.

Achievement System

Recognition that drives progress

Motivation through measurable achievement. Badges, contribution tracking, learning streaks, and verifiable certificates provide tangible recognition for every milestone reached.

Five Badge Types

Badges are awarded automatically when conditions are met: consecutive daily activity, course completion sets, time-constrained combos, certificate attainment, and milestone events. Administrators can create, toggle, or manually award any badge.

Contribution Tracking

A GitHub-style activity heatmap on every profile captures 365 days of learning. Each content completion logs a day of engagement, rendered with daily colour intensity and streak counters that quantify consistency.

Verifiable Certificates

Administrators design reusable certificate templates with header, subtitle, description, issuer, and footer. Certificates are awarded by category and subcategory, then displayed on the recipient's profile for sharing.

Administration

Complete institutional control

A comprehensive administration panel provides oversight across every aspect of the platform. Manage content, users, submissions, certificates, badges, and uploaded files from a unified interface.

Content Management

Visual course tree with capture and rebuild, Markdown preview and editing, batch metadata updates, and item locking with Pass, Date, and Manual lock types.

User Administration

Comprehensive user listing with role assignment. Ban management with configurable duration (days, weeks, months) and reason logging. Unban capability restores access.

Submission Review

Instructors review project code submissions and issue verdicts: Passed to mark completion, or Retry with feedback for revision. Supports re-submission on retry.

Certificate Templates

Full CRUD for certificate designs. Administrators define reusable templates and award them to users by category and subcategory. Certificates appear on recipient profiles.

Badge Administration

Create, edit, enable, and disable badge definitions. Toggling a badge off prevents automatic awarding. Manual award capability for administrative recognition.

File and Image Upload

Upload images and files (PNG, JPG, GIF, SVG, WebP) to a dedicated directory. Referenced directly from course content and administrative pages.

Platform Capabilities

Designed for real-world education

Role-Based Access

Three roles — student, instructor, administrator — with scoped permissions. Server-side sessions secured with industry-standard password hashing.

Prerequisite Gating

Content can require prior completion. Three lock types: Pass (completion unlocks next), Date (scheduled, absolute or relative), and Manual (administrator toggle).

Progress API

RESTful progress tracking across all content types. Supports step saving, content completion marking, project submission, instructor verdict workflow, and re-submission on retry.

Card-Based Interface

Wide cards with proportional scaling for deeper categories. No file-system icons or expandable tree widgets — a clean, focused browsing experience with breadcrumb navigation.

Theme Support

Full dark and light mode via CSS custom properties. All surfaces use tinted shades — no pure white or black — for reduced eye strain across different environments.

Dashboard and Profile

Personal dashboard with completed content summary. Profile page featuring avatar, bio, earned badges, awarded certificates, and the contribution activity graph.

Architecture

Clean separation, scalable by design

DigitalEdu follows a modular blueprint architecture with distinct route, service, and data layers. The filesystem serves as the authoritative source for course structure, while a database handles user state and achievements.

Route Layer

Five Flask blueprints organise all HTTP endpoints: authentication, course browsing, progress tracking via RESTful API, administration, and general pages. Each blueprint is independently maintainable.

Service Layer

The course parser scans the filesystem and parses YAML front-matter into a structured curriculum. The assessment parser extracts questions, workshop steps, and practical requirements from Markdown content.

Data Layer

Eight SQLAlchemy ORM models: User, Progress, Badge, UserBadge, ActivityLog, Ban, CertificateTemplate, and Certificate. SQLite by default for zero-configuration deployment; PostgreSQL-ready through a single environment variable change.
